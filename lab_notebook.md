# Lab Notebook

This is a chronological record of everything done in this research project — decisions made, analyses run, things tried that didn't work, unexpected findings, and changes in direction. It is the anti-cherry-picking mechanism.

The lab notebook cannot be backdated. Entries are dated when written. If an entry is added retrospectively, it is clearly marked as such with an explanation of why.

An analyst who reads this notebook from beginning to end should be able to reconstruct exactly how the research unfolded — including the dead ends.

---

## Format

```
### YYYY-MM-DD — [Short description of what happened]

**What was done**: [Specific actions taken]
**Why**: [Reasoning or decision]
**What was found**: [Outcome, including null/negative outcomes]
**Implications**: [Does this change anything in the protocol? Does it generate a hypothesis?]
**Files created/modified**: [Paths]
**Open questions raised**: [Anything this entry leaves unresolved]
```
